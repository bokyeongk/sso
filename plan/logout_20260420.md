# 로그아웃 구현 플랜

**작성일**: 2026-04-20  
**기능**: SSO 통합 로그아웃

---

## 요구사항

1. 화면 로그아웃 버튼 클릭 시 로그아웃 API 호출 (refreshToken 포함)
2. 백엔드 Keycloak 로그아웃 API 호출
3. 백엔드 쿠키 만료 처리 (access_token, refresh_token 쿠키 삭제)
4. 프론트 응답 수신 후 로그인 페이지(`/login`)로 이동

---

## 현재 구현 현황 분석

| 항목 | 현재 상태 | 변경 필요 |
|------|-----------|-----------|
| `POST /api/auth/logout` 엔드포인트 | ✅ 존재 | refreshToken 쿠키 읽기 확인 필요 |
| Keycloak 로그아웃 API 호출 | ✅ `AuthApplicationService.logout(refreshToken)` | 파라미터 전달 확인 |
| 쿠키 삭제 (`clearTokenCookies`) | ✅ maxAge=0 처리 | 정상 동작 확인 |
| 프론트 로그아웃 후 이동 경로 | ❌ `/` (홈)으로 이동 | `/login`으로 변경 |
| refreshToken을 요청 바디에 포함 | 확인 필요 | 쿠키 자동 전송으로 대체 가능 |

## 구현 작업

### 1단계: DB (해당 없음 — 스킵)

### 2단계: 백엔드 (`backend-architect` 에이전트)

#### 2-1. `AuthController.logout()` 확인 및 수정
- 파일: `backend/src/main/java/com/hubilon/sso/adapter/in/web/AuthController.java`
- `POST /api/auth/logout` 엔드포인트
- `@CookieValue("refresh_token")` 로 refresh_token 쿠키 읽기
- `authApplicationService.logout(refreshToken)` 호출
- `clearTokenCookies()` 호출 (access_token, refresh_token 쿠키 maxAge=0)
- 성공 응답 반환 (200 OK)
- SecurityConfig에 `/api/auth/logout` 공개 경로 등록 확인 (`permitAll`)

#### 2-2. `AuthApplicationService.logout()` 확인 및 수정
- 파일: `backend/src/main/java/com/hubilon/sso/application/service/AuthApplicationService.java`
- Keycloak logout endpoint: `POST {keycloak.url}/realms/{realm}/protocol/openid-connect/logout`
- 요청 파라미터:
  - `client_id`: Keycloak 클라이언트 ID
  - `refresh_token`: 전달받은 refresh token
- RestTemplate으로 POST 요청 (form-urlencoded)
- Keycloak 응답 실패 시에도 쿠키 삭제는 수행 (예외 처리)

### 3단계: 프론트엔드 (`frontend-architect` 에이전트)

#### 3-1. `lib/auth.ts` — `startLogout()` 수정
- 파일: `frontend/src/lib/auth.ts`
- `POST /api/auth/logout` 호출 (withCredentials: true — 쿠키 자동 포함)
- refreshToken은 쿠키로 자동 전송 (별도 바디 불필요)

#### 3-2. `hooks/useAuth.ts` — logout 후 이동 경로 수정
- 파일: `frontend/src/hooks/useAuth.ts`
- 로그아웃 완료 후 `/login`으로 이동
- 현재: `window.location.href = '/'` → 변경: `window.location.href = '/login'`

## 흐름 다이어그램

```
[홈페이지 로그아웃 버튼 클릭]
        ↓
[useAuth.logout() 호출]
        ↓
[POST /api/auth/logout 호출]
[  쿠키: refresh_token 자동 포함  ]
        ↓
[백엔드 AuthController.logout()]
        ├─ Keycloak logout API 호출 (refresh_token 전달)
        └─ 쿠키 삭제 (access_token, refresh_token maxAge=0)
        ↓
[200 OK 응답]
        ↓
[authStore.setAuthenticated(false)]
        ↓
[window.location.href = '/login']
```

## 리뷰 이슈 반영 (2026-04-20)

### 보안

| 심각도 | 이슈 | 처리 방향 |
|--------|------|-----------|
| **높음** | CSRF 공격 가능 — `permitAll` 설정 시 외부 사이트에서 강제 로그아웃 가능 | 기존 쿠키 `SameSite=Strict` 설정으로 방어. SecurityConfig에서 CSRF 보호 유지 여부 확인 |
| 중간 | `@CookieValue("refresh_token")` 쿠키 없을 때 400/NPE 발생 가능 | `required = false`로 설정. 쿠키 없으면 Keycloak 호출 생략하고 쿠키 삭제만 수행 후 200 반환 |
| 중간 | Keycloak 로그아웃 실패 시 서버 세션 유지 | Keycloak 호출 실패해도 쿠키 삭제는 반드시 수행. 클라이언트 세션은 종료됨을 허용 |
| 낮음 | confidential 클라이언트인 경우 `client_secret` 필요 | Keycloak 클라이언트 타입 확인 후 필요 시 `client_secret` 추가 |

### 리팩토링

| 심각도 | 이슈 | 처리 방향 |
|--------|------|-----------|
| 중간 | `clearTokenCookies()` 중복 구현 방지 | `AuthController` 내 기존 공통 메서드 재사용 확인 후 중복 작성 금지 |
| 낮음 | `window.location.href` 하드코딩 | 프론트 라우터(`react-router`) 사용 여부 확인 후 `navigate('/login')` 교체 검토 |

### 기능

| 심각도 | 이슈 | 처리 방향 |
|--------|------|-----------|
| **높음** | 로그아웃 API 호출 실패 시 프론트 처리 미정의 | API 실패(네트워크 오류, 5xx) 시에도 `authStore.setAuthenticated(false)` 후 `/login` 이동 |
| 중간 | `authStore.setAuthenticated(false)` 타이밍 | API 성공/실패 모두 `finally` 블록에서 상태 초기화 및 `/login` 이동 |
| 낮음 | 쿠키 없는 상태에서 중복 로그아웃 요청 | 백엔드에서 쿠키 없으면 바로 200 반환. 프론트에서 `isAuthenticated` 체크로 중복 방지 |
