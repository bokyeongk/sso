# /bk_test — 프로세스 실행

사용자가 `/bk_test <plan명>` 을 입력하면 아래 프로세스를 순서대로 실행한다.

## 사전 확인: plan 파일 탐색

1. `plan/` 디렉토리에서 `<plan명>_*.md` 패턴에 매칭되는 파일을 찾는다.
    - 매칭 파일이 여러 개면 날짜 기준 가장 최신 파일을 선택한다.
    - 매칭 파일이 없으면 사용자에게 알리고 중단한다:
      > "plan/<plan명>_yyyymmdd.md 파일을 찾을 수 없습니다. plan 파일명을 확인해 주세요."

---

## Test Agent — 테스트 실행

Test Agent(general-purpose)에게 테스트를 지시한다.

- plan.md를 읽어 구현 범위 파악, 테스트 없으면 작성 후 실행
- 백엔드: `./gradlew test` (build.gradle 없으면 사용자에게 알리고 중단)
- 프론트엔드: `npm run test -- --run` (test 스크립트 없으면 사용자에게 알리고 중단)
- 실패 시 구현 Agent에게 수정 지시 후 재시도 (최대 3회)
- 3회 초과 시 실패 내용 사용자에게 보고 후 중단
- 성공 시: "✅ <plan명> 구현 및 테스트 완료"

---

## 주의사항

- 사용자 확인이 필요한 시점에서는 반드시 대기한다.
- 모든 Agent 지시는 한국어로 구체적으로 작성한다.
