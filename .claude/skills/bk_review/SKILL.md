# /bk_review — 프로세스 실행

사용자가 `/bk_review <plan명>` 을 입력하면 아래 프로세스를 순서대로 실행한다.

## 사전 확인: plan 파일 탐색

1. `plan/` 디렉토리에서 `<plan명>_*.md` 패턴에 매칭되는 파일을 찾는다.
   - 매칭 파일이 여러 개면 날짜 기준 가장 최신 파일을 선택한다.
   - 매칭 파일이 없으면 사용자에게 알리고 중단한다:
     > "plan/<plan명>_yyyymmdd.md 파일을 찾을 수 없습니다. plan 파일명을 확인해 주세요."

---

## Review Agent — plan 검토

선택된 plan 파일을 Review Agent(general-purpose)에게 전달하여 다음 항목을 검토하도록 지시한다:

- **보안**: SQL Injection, XSS, 인증/인가 누락, 민감정보 노출 여부
- **리팩토링**: 코드 중복 가능성, 불필요한 복잡도, 공통 추상화 필요 여부
- **기능**: 요구사항 누락, 엣지케이스, API 스펙 일관성

검토 완료 후:
- 이슈 발견 → 사용자에게 이슈 목록 보고 후 대기. 사용자 승인 시 계속 진행.
- 이상 없음 → plan.md 하단에 아래 형식으로 리뷰 결과를 기록 후 2단계 진행:

```
## Review 결과
- 검토일: yyyy-mm-dd
- 검토 항목: 보안 / 리팩토링 / 기능
- 결과: 이상 없음
```

## 주의사항

- 모든 Agent 지시는 한국어로 구체적으로 작성한다.
